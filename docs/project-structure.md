# Project Structure

CampusOps AI is split into clear human-facing folders.

```txt
CampusOps/
  frontend/
    src/
      components/        UI modules and role workspaces
        actions/         Admin/faculty daily Action Center
        automation/      Admin Automation Control Center
        imports/         Admin CSV/XLSX import center
        knowledge/       Policy Knowledge and RAG Center
        reports/         Admin/faculty reports and export center
      data/              Seed data used by the frontend demo paths
      lib/               Validation, API, routing, persistence helpers
      App.tsx            Main app shell and role navigation
      App.css            Shared UI styling
      types.ts           Shared frontend domain types
    public/              Static icons and assets
    package.json         Frontend scripts and dependencies
    vite.config.ts       Vite + API proxy configuration

  backend/
    src/
      server.js          HTTP API routes
      db.js              SQLite schema, seed data, and persistence helpers
      validation.js      Backend validation and duplicate checks
    data/
      campusops.sqlite   Local SQLite database, generated automatically
    package.json         Backend scripts

  docs/
    architecture.md
    project-structure.md
    demo-script.md

  scripts/
    dev.mjs              Starts backend and frontend together

  README.md              Project overview and run instructions
```

## What To Ignore

These are generated by tools and are not project logic:

- `frontend/node_modules/`
- `frontend/dist/`
- `backend/data/*.sqlite-*`
- `.vite/`
